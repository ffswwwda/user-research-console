        核心算法 · 每时间步执行的五个步骤
        引擎对 0 个智能体、共 12 个时间步，逐步执行以下循环。这是对 OASIS env.step() 主循环的忠实简化实现。
        // 0. 初始化：人格生成 + 社交网络（偏好依附 + 同质性）
for each agent i:
    x_i   ← 立场/态度 ∈ [-1,+1]      (按人格群落先验采样)
    ε_i   ← 有界信心阈值 ∈ [0.15,0.6] (openness, 越大越易被说服)
    a_i   ← 活跃度 ∈ [0,1]           (决定被激活概率)
    w_i   ← 影响力 ∝ 粉丝数 (+KOL加成)
followees_i ← 依附概率 P(j) ∝ degree_j + 同质性(|x_i-x_j|越小越易关注)

for t = 0 … T:  // 逐时间步社会演化
  1. 激活 Activation
     arousal(t) = e^(-t/4)                       // 事件初期更亢奋
     P_active(i) = a_i · (0.35 + 0.65·arousal) · (1 - 0.25·信息公开度)
  2. 推荐 Recommendation（Reddit hot-score + 二维嵌入相似度）
     hot(p)  = sign(ups-downs)·log10(max(|ups-downs|,1)+1) + 0.35·created(p)
     sim(a,p)= 0.65·(1 - |x_i - stance(p)|/2) + 0.35·(1 - |y_i - y_p|/2)  // 立场一致 + 潜在维度一致
     rank(p) = hot(p) + 0.8·sim(a,p)                         // 个性化加权 → 制造信息茧房
     feed_i  = TopK(rank, K=5) 取自 followees 的帖子 + 全局热帖
  3. 动作决策 Action（规则策略替代 LLM 推理）
     agree = 1 - |x_i - stance(p)|/2
     agree>1-ε → LIKE / REPOST(放大)  ·  agree<ε → DISLIKE / COMMENT(反驳)  ·  其余 → do_nothing
     以概率 a_i·(0.2+0.5·|x_i|) 发帖 CREATE_POST，立场≈x_i
     高认同且作者有影响力 → FOLLOW
  4. 观点更新 Opinion Dynamics（有界信心 + 社会影响 DeGroot）
     对 feed 中满足 |x_i - stance(p)| < ε_i 的帖子：
        x_i ← x_i + μ · Σ w_p·(stance(p) - x_i) / Σ w_p     μ = 0.25·(1+0.5·KOL介入)
     外生"上帝变量"注入：
        x_i += 信息公开度·0.015·(理性基线 - x_i)
        若群落先验为负: x_i += 补偿力度·0.035·(0.6 - x_i)
     x_i ← clip(x_i, -1, +1)
  5. 记忆 & 统计 Memory（Zep episode 同步）
     记录动作计数、更新帖子 ups/downs、老化清理 3 步前的帖子
     态度分布 → 正面(x>0.15) / 中性 / 负面(x<-0.15)，方差=极化度
        逐时间步运行轨迹
        下表由引擎实际计算得出，每一行是一个时间步的真实统计。
        T激活发帖点赞转发评论均值 x̄负面%
        动作空间分布随时间演化
        OASIS 动作空间在本模型中的实际触发量（发帖/点赞/转发/评论）。
        蒙特卡洛稳定性 · 终态均值意见的分布
        同一种网络、不同随机流重复仿真 N 次，终态群体均值意见的分布。分布越窄、标准差越小，推演结论越稳健可靠（统计显著性越高）。
        关键实现说明
        · 确定性可复算 + 蒙特卡洛：引擎用种子化伪随机数（mulberry32），单跑时同参数结果一致、调"上帝变量"后的差异完全可归因到变量本身；打开蒙特卡洛开关后会用不同随机流重复仿真 N 次，输出均值 ± 95% 置信区间与终态分布直方图，让结论带统计显著性。
        · 被替代的部分：真实 MiroFish 里第 3 步的动作决策与第 5 步的发帖文本由 LLM（Qwen-plus / GPT-4o）生成；此处用基于立场差 + 人格特质的规则化策略近似，保留决策的逻辑结构，去掉需要大模型的语言生成。
        · 忠实保留且已深化：激活机制、Reddit hot-score 推荐排序、动作空间选择逻辑、有界信心 + DeGroot 社会影响的观点动力学、Zep 式记忆计数——都直接实现；并新增二维嵌入相似度推荐（立场 + 潜在维度，制造信息茧房）与三种网络拓扑（无标度/小世界/随机）可选。
        · 干预变量：卖点透明度 / 痛点解决力度 / 达人·口碑投放（语义随场景切换）是作为外生力项注入到第 1、4 步的真实参数；拖动滑杆会真正重跑数值引擎并重算全部曲线（MC 模式下为节省实时算力，拖动时用较小样本，点"重新构建"得到完整样本）。

        如何把你的用户数据接进来
        基座当前为研究原型的内置示例分群。要换成你真实的用户研究数据，只需提供一份结构化导出（VOC 聚类结果 / 类目洞察分群），按下方 schema 替换即可——引擎逻辑不变，换的是"人格来源"。
        { "segments": [ { "name": "价格敏感者", "weight": 0.18, "x": -0.15, // 态度先验 "y": -0.70, // 价格—品质取向 "act": 0.55, // 活跃度 0–1 "advocate": false, // 是否倡导者/KOL "tags": ["用户画像:价格敏感","动机:性价比","阻碍:预算有限"] } ] }
        与你的研究体系对齐
        这张基座表直接对接你已经建好的两套工具：
        • voc-report：九大标签模块（用户画像分群 / 购买动机 / 使用场景 / 使用阻碍 / 忠诚度 / 改进建议 / 十三种需求）→ 决定每个分群的 x、y 与痛点。
        • category-insight-hub：市场细分、JTBD、需求四象限、价格带 → 决定分群构成、需求轴与干预杠杆。
        下一步可做的：把 VOC 聚类结果一键导出为该 schema，让真实分群驱动仿真，而非内置示例。
        导入真实分群（覆盖内置示例）
        把 voc-report / category-insight-hub 导出的分群 JSON 粘贴进来，前端用确定性采样按真实分布生成人格，不依赖大模型。schema：{segments:[{name,weight,x,y,act,advocate,tags}]}
        社媒真实数据基座（content-analytics-dashboard）
        当场景选「社媒活动预测」时，引擎会用这里的真实社媒基准来标定预测——爆款率、互动率、账号矩阵、最佳发布时间、活动提升系数。默认已内置 insights.json 的真实聚合值；把你的 content_data.json 原样粘贴进来，前端会自动重算全部基准（无需大模型）。

        架构说明

        MiroFish 构建于 CAMEL-AI 开源的 OASIS 社会仿真引擎之上，融合 GraphRAG 图谱与 Zep 长期记忆，形成完整的"提取 → 建模 → 演化 → 报告 → 交互"闭环。

        🧬

        种子提取 & GraphRAG

        从新闻、政策、金融信号、小说文本中抽取实体与关系，构建高保真知识图谱，为每个智能体注入个体记忆与集体记忆。

        🎭

        人格生成引擎

        基于图谱实体自动生成拥有独立性格、立场、行为逻辑的智能体，并完成 Agent 配置注入，构成一个可信的数字社会。

        🌊

        双平台并行仿真

        底层基于 OASIS 引擎，支持双平台并行模拟，动态更新时间记忆，让数千智能体自由互动、社会演化。

        📄

        ReportAgent

        携带丰富工具集，在仿真结束后与数字世界深度交互，自动解析预测需求，产出可读的结构化预测报告。

        💬

        深度交互层

        用户可与世界中任意智能体对话，也可与 ReportAgent 交互追问，把黑箱预测变成可对话、可审视的过程。

        🌍

        God's-Eye 变量注入

        从"上帝视角"动态注入变量，精确推演不同干预下的未来轨迹——这是决策排练的核心能力。
